## 题目描述

众所周知，**时空壶（Timekettle）** 是一个研发专业翻译耳机和翻译设备的智能硬件公司。事实上，我们同时也是一家软件公司，我们绝大多数软硬件产品都非常依赖于后端云平台服务提供稳定的 RTC 网络服务和基于语言的大模型翻译的编排能力。

假设你是云平台的开发者，我们现在希望设计一个**简易版的同声传译 Agent**，以此来实现多种语言的翻译能力。

**Agent 时代最好的工具其实是 CLI**，因此我们希望这个 Agent 可以用 CLI 模式启动。我们希望这个 Agent 具备如下两种运行模式：

### 模式一：流式同传模式

可用如下命令用 `mini-tmk-agent` 启动一个可流式进行中翻英的 Agent：

```bash
mini-tmk-agent stream --source-lang zh --target-lang en
```

此时 Agent 将进行**持续监听电脑麦克风并进行流式翻译**：一旦检测到有源目标语言输入，会实时在终端上同时显示**源目标语言识别后的文字**和**翻译后目标语言的文字**。翻译的语种至少支持**中英西日** 4 种语言。

### 模式二：基于文件转录模式

此外，我们也希望 Agent 支持 **Transcript** 模式，比如我们可以给 Agent 提供一个**本地目标语音文件**（比如 PCM、WAV 或者 MP3 格式），它可在**本地生成转写好的文本文件**。比如：

```bash
# 用 mini-tmk-agent 启动 Transcript 模式
mini-tmk-agent transcript --file <your-audio-file> --output <destination-file-path>
```

## 编码要求

1. 整个项目的完成度足够高，**有规范的设计和使用文档、单元测试和集成测试等**，代码足够简洁干净，README 要能确保绝大多数人都能开箱即用，运行环境尽量能以 **Linux 或者 macOS 优先**；
2. 你可以使用任何你觉得顺手的 AI Coding 工具，哪怕整个项目所有代码都是 AI 写的都没关系，**但我们希望你为人类能够脱机解释每一行代码并完整叙述整个开发过程的所有细节**，这个项目中的每一行代码都有可能成为面试考核的一部分；
3. 实现过程中，你可以任选你喜欢的 LLM 作为你的 ASR 和 Translation 模型（当然，你也可以用端到端的模型），我们对翻译的准确度不做太高的要求。你可以使用类似于 OpenRouter 或者硅基流动这种聚合类的服务来快速接入大模型，也可以直接选择 OpenAI、千问、豆包、Gemini、MiniMax 等对应大模型厂商的模型；
4. **必须采用 Go 语言完整实现功能**（如果你是其他语言的开发者，正好可以借这个机会在大模型的帮助下速通 Go 语言）；
5. **必须将完整的代码提交到 GitHub 上**，面试时你可完整演示从代码下载、编译构建和运行的所有过程；
6. 请在拿到题目的同时 **3 天内完成** 所有的编码工作（具体时长可酌情调整，仅作为其中一个参考因素）；

## 加分项目（可基于个人能力实现一项或多项）

1. 实现 **TTS 输出**，即不仅有文字输出，也有目标语言的输出；
2. 可实现基于 RTC 网络的 Agent 之间的通信，比如可以在两个 Terminal 之间（也可以是不同的机器上）实现：A 终端说话，B 终端可接收到翻译后的语言文本或者 TTS。关于 RTC 网络的选型，可选择**火山引擎 RTC** 或者 **声网 RTC**（这两家都有足够完成试验的免费额度）；
3. **预定规定时间更快**（比如几个小时内）完成几乎所有功能；
4. 能为你的 Agent 实现一个**简易好看的 Web UI**；
5. **任何**你觉得可以展示你个人技术实力的创意；
